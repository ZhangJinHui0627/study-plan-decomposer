# iOS 与 Android 对齐设计方案

## 范围与边界

- 仅修改 `Study-plan-iOS` 目录。
- `Study-plan- Android` 目录及其全部代码、资源、构建文件和测试保持只读。
- 页面结构、信息层级、文案、颜色和间距尽量 1:1 对齐 Android。
- 导航、返回、滑动、系统键盘、系统分享和弹窗使用 iOS 原生 SwiftUI 交互。
- 不添加 Android 没有的业务功能，不引入第三方依赖，不做无关架构重构。
- 保留现有 Xcode 用户界面状态文件的改动，不删除 `.DS_Store`、`.derivedData` 等隐藏或生成文件。

## 实现边界与数据流

- `StudyPlanStore` 继续作为页面唯一状态入口；页面不直接写 `UserDefaults`、通知或日历。
- `TimerEngine` 负责计时状态与生命周期；Store 负责今日累计时长、持久化、通知和任务排序。
- 保留 SwiftUI 的 `NavigationStack`、`List`、`Form`、`sheet`、`confirmationDialog` 与 `swipeActions`。
- AppIcon 只修改 iOS 资源，源图使用 Android 现有 `app_icon.jpg`。

## 页面改造

### 主页

- 对齐 Android 的任务卡片间距、头像、标题/预览、日期和展开详情布局。
- 保留中央“点击完成”胶囊，并与手动打卡开关联动。
- 批量删除时显示底部操作条、禁用拖拽，并保留 iOS 原生多选与确认弹窗。
- 保留 iOS 左右滑动删除/完成及长按菜单。
- 统一新增、编辑、快速拆解、搜索的字段顺序、默认值和文案。

### 计时

- 对齐 Android 的 220pt 计时圆环、160×50pt 主按钮和任务/今日累计卡片。
- 使用 iOS sheet 实现任务选择和计时选项。
- 支持倒计时、正计时、溢出计时、任务时长为 0 的拦截、运行中切换确认及结束后的完成选择。
- 保留运行中的全屏流光边框和 iOS 原生震动/通知能力。

### 统计

- 按 Android 顺序呈现三列统计、学习小结、专注建议、任务状态概览和学习进度。
- 完成任务的计划时长和页数计入统计；完成率与任务状态同步刷新。

### 个人中心与设置

- 使用 Android 的分组卡片层级呈现资料、提醒、震动、默认计时、词库、导出和清空任务。
- 保留 PhotosPicker、ShareLink、NavigationStack 等 iOS 原生能力。
- 设置页提醒时间与任务提醒行为保持一致，修改后重新安排已有任务提醒。

## 逻辑修复

1. 计时暂停、继续、切换任务或停止时，累计基准跟随当前已记录秒数，避免暂停后重复累计。
2. 应用切后台或重新打开时，根据 `startedAt`、`pausedAt` 和当前时间恢复 elapsed；已结束的计时不重复触发结束处理。
3. 倒计时开始前取消旧通知；暂停、停止或切换任务时取消；恢复时按剩余时间重新安排；正计时不安排结束通知。
4. 任务提醒继续使用 `specificTime`；提醒开关关闭时取消全部待办提醒，开启时重新安排。
5. 手动拖拽后的顺序在新增、编辑、删除和重启后保持；自动排序只在明确需要时恢复，计时任务置顶不破坏手动顺序数据。
6. 计时完成、手动完成、重做任务时，同步任务卡片、统计、日历事件和提醒。
7. 清空任务时取消通知并删除对应日历事件。

## 图标资源

- 将 Android 的 `Study-plan- Android/app/src/main/res/drawable/app_icon.jpg` 作为 iOS AppIcon 的 1024×1024 源图。
- 补齐 `Study-plan-iOS/Study-plan/Assets.xcassets/AppIcon.appiconset/Contents.json` 的文件引用。
- 不修改 Android 图标资源。

## 验收标准

- iOS 项目可成功编译。
- Android 目录没有任何 Git diff。
- iOS AppIcon 资源配置包含实际源图。
- 计时暂停后继续，今日累计时长不重复增加。
- 计时切后台再回到前台，显示时间与真实经过时间一致。
- 任务提醒、完成、重做、排序、删除和清空流程可重复执行且状态不丢失。
- 保留现有 Xcode 用户界面状态文件改动，不删除隐藏或生成文件。
